상태머신

트리거는 딱 2종 — 탭(사용자가 우리 버튼을 누름)과 도착 시(외부 T맵에서 앱으로 돌아오는 이벤트). 핵심 규칙: 버튼 탭이 곧 전진 확정이다. 외부 T맵의 ‘안내’ 완료를 기다리지 않는다.

탭 사용자 버튼 도착 시 T맵 복귀 이벤트 외부 T맵(딥링크/Intent) 완료 done 누적

참고 구현 — reduce()

구조는 팀 아키텍처(MVI/ViewModel 등)에 맞게 조정. 이름·분기·순서는 프로토 상태머신을 그대로 옮긴 것.

RideStateMachine.ktKotlin
enum class Screen { PickupGo, TmapSel, PtmapNav, PickupArrived,
                    Preview, DriveGo, TmapNav, DriveArrived, Pay, Done, FullRoute }

data class Ride(
  val screen: Screen,
  val target: Int,                 // 현재 포커스 스톱 (0=고객, 1=경유1 … last=도착지)
  val done: Set<Int> = emptySet(),
  val onTrip: Boolean = false,     // 픽업 완료 후 '운행' 구간 여부
)

sealed interface Ev {
  data object DepartToCustomer : Ev  // 고객 위치로 출발하기
  data object Guide : Ev             // 안내 (T맵 딥링크 실행)
  data object ArrivedBack : Ev       // T맵에서 도착 후 앱 복귀
  data object NotifyArrived : Ev     // 고객에게 도착 알리기
  data object StartDrive : Ev        // 운행 시작하기
  data object GetRoute : Ev          // 경로 안내받기
  data object StopDone : Ev          // 경유지 도착 완료
  data object Pay : Ev               // 요금 결제 / 결제하기
  data object ViewAll : Ev           // 전체 보기
  data object NavFromFull : Ev       // 전체 경로에서 정류장 내비 안내
}

// 버튼 탭 = 전진 확정. 외부 T맵의 '안내'를 기다리지 않는다.
fun Ride.reduce(e: Ev): Ride = when (e) {
  Ev.DepartToCustomer -> copy(screen = Screen.TmapSel, target = 0, onTrip = false)
  Ev.Guide            -> copy(screen = if (onTrip) Screen.TmapNav else Screen.PtmapNav)
  Ev.ArrivedBack      -> if (onTrip) copy(screen = Screen.DriveArrived, done = done + target)
                         else        copy(screen = Screen.PickupArrived)
  Ev.NotifyArrived    -> copy(screen = Screen.Preview)
  Ev.StartDrive       -> copy(screen = Screen.DriveGo, target = 1, onTrip = true) // 경유1 직행
  Ev.GetRoute         -> copy(screen = Screen.TmapSel)          // onTrip 유지 → Guide가 TmapNav로
  Ev.StopDone         -> copy(screen = Screen.DriveGo, target = target + 1) // 다음 스톱
  Ev.Pay              -> if (screen == Screen.Pay) copy(screen = Screen.Done)
                         else copy(screen = Screen.Pay)
  Ev.ViewAll          -> copy(screen = Screen.FullRoute)
  Ev.NavFromFull      -> copy(screen = Screen.TmapSel)     // 전체 경로 닫힘 · 복귀 navReturn=drive
}
T맵 핸드오프 = 복원 불가. T맵은 딥링크/Intent로 여는 외부 앱이라 콜백이 없다. “취소 시 이전 상태 복원”은 불가 — 전진은 우리 버튼 탭 기준으로만 확정한다.
전체 경로 = 일시적 오버레이. 전체 경로(full)에서 정류장 내비 안내를 탭하면 전체 경로가 닫히고 T맵으로 전환. T맵 복귀(도착·취소)는 전체 경로가 아닌 드라이버 메인(drive)으로 돌아온다. (프로토: peekGo의 navReturn='drive')

참고 구현 — T맵 핸드오프 (딥링크 · 복귀 감지)

외부 T맵은 콜백이 없다 → 딥링크 Intent로 실행하고, 복귀는 onResume로만 감지. 전진 확정은 T맵이 아니라 우리 버튼 탭 기준.

TmapHandoff.ktIntent
// T맵 경로안내 딥링크 실행 (스킴·파라미터는 T맵 SDK 문서로 확인)
fun launchTmap(ctx: Context, stop: Stop) {
  val uri = Uri.parse(
    "tmap://route?goalname=${Uri.encode(stop.name)}" +
    "&goalx=${stop.lng}&goaly=${stop.lat}"      // x=경도, y=위도
  )
  val intent = Intent(Intent.ACTION_VIEW, uri)
    .addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
  if (intent.resolveActivity(ctx.packageManager) != null) ctx.startActivity(intent)
  else openPlayStore(ctx, "com.skt.tmap.ku")          // 미설치 → 스토어
}

// 복귀 감지: T맵은 콜백이 없다 → onResume 로 '돌아옴'만 안다.
// 도착·전진은 T맵이 아니라 우리 버튼 탭으로 확정 (복원 불가 규칙).
override fun onResume() {
  super.onResume()
  if (state.screen in setOf(Screen.TmapNav, Screen.PtmapNav))
    viewModel.onReturnedFromTmap()
}

참고 구현 — 상태별 색 (배지 · 마커)

완료=물러남, 현재 타깃만 시선 독점. 배지(리스트)·마커(지도)가 같은 규칙을 써서 상태색이 항상 일치한다. Figma엔 색만 있고 “언제 어떤 색”인지 분기 로직은 없다.

StopColors.ktCompose
// 상태 → 색. 배지(리스트)·마커(지도) 공통 → 항상 일치.
enum class StopVisual { Pending, Focus, Done }

val Navy700 = Color(0xFF293F71)   // 대기·완료
val Blue700 = Color(0xFF0074D7)   // 현재 타깃(경유·출발)
val TadaRed = Color(0xFFFE4C00)   // 현재 타깃(도착지)
val DoneGrn = Color(0xFF30A234)   // 리스트 "완료" 텍스트

fun stopColor(v: StopVisual, isDest: Boolean): Color = when (v) {
  StopVisual.Focus -> if (isDest) TadaRed else Blue700   // 현재만 파랑/빨강
  StopVisual.Pending, StopVisual.Done -> Navy700         // 대기·완료 = navy
}

// 완료 = 물러남: 지도 핀 dim(alpha .5)+✓, 리스트는 초록 "완료".
@Composable
fun MapPin(v: StopVisual, isDest: Boolean) = Pin(
  tint  = stopColor(v, isDest),
  alpha = if (v == StopVisual.Done) 0.5f else 1f,
  check = v == StopVisual.Done,
)

Motion.ktCompose
// 프레스: scale + ripple, transition ≈ 110ms. scale 값만 바꿔 요소별 대응.
@Composable
fun PressScale(scale: Float = 0.98f, onClick: () -> Unit, content: @Composable () -> Unit) {
  val interaction = remember { MutableInteractionSource() }
  val pressed by interaction.collectIsPressedAsState()
  val s by animateFloatAsState(if (pressed) scale else 1f, tween(110), label = "press")
  Box(
    Modifier
      .graphicsLayer { scaleX = s; scaleY = s }
      .clickable(interaction, indication = ripple(), onClick = onClick),
    content = { content() },
  )
} // 큰 버튼 0.97 · 배지/버블 0.91 · 내비 화살표 0.74 · 닫기(✕) 0.86

val Emphasized = CubicBezierEasing(0.2f, 0.7f, 0.3f, 1f)

// 도착 알림 → preview: 항목이 아래(16dp)에서 순차 등장. 지연 = index × 35ms.
@Composable
fun StaggerIn(index: Int, content: @Composable () -> Unit) {
  var shown by remember { mutableStateOf(false) }
  LaunchedEffect(Unit) { delay(index * 35L); shown = true }
  val p by animateFloatAsState(
    if (shown) 1f else 0f, tween(300, easing = Emphasized), label = "stagger",
  )
  Box(Modifier.graphicsLayer { alpha = p; translationY = (1f - p) * 16.dp.toPx() }) {
    content()
  }
}

val Standard = CubicBezierEasing(0.65f, 0f, 0.35f, 1f)

// preview → drive 카드 모프: 이름 축소 + 색 dim + 높이 동시(≈580ms).
// 여러 속성 동시 → MotionLayout(ConstraintSet 2개) 권장. 아래는 updateTransition 대안.
val t = updateTransition(ride.screen, label = "cardMorph")
val nameScale by t.animateFloat({ tween(580, easing = Standard) }, label = "scale") {
  if (it == Screen.DriveGo) 0.769f else 1f          // 이름 축소
}
val nameColor by t.animateColor({ tween(580, easing = Standard) }, label = "color") {
  if (it == Screen.DriveGo) NavyDim else White      // 색 dim
}
val cardHeight by t.animateDp({ tween(580, easing = Standard) }, label = "height") {
  if (it == Screen.DriveGo) driveCardH else previewCardH  // 높이 변화
}

Responsive · 큰 글씨 대응

반응형 (sp/dp 댐핑)

콘텐츠 텍스트는 sp(base 18sp), OS·외부(T맵)는 dp. 큰 글씨 접근성 설정에서 레이아웃이 깨지지 않도록, 내부 패딩·갭은 폰트배율을 완만히(댐핑) 따라가고 요소 간 마진은 dp로 고정한다.

Scaling.ktCompose
// 콘텐츠 = sp. 내부 패딩·갭은 폰트배율 '댐핑', 요소 간 마진은 dp 고정.
val fs  = LocalConfiguration.current.fontScale     // 1.0 = 기본
val fsx = 1f + (fs - 1f) * 0.4f                     // 패딩·갭 댐핑 계수
val innerPadding     = (16 * fsx).dp                 // 내부 패딩: sp 비례로 완만히
val elementMargin    = 12.dp                          // 요소 간 마진: 고정 dp
val sheetMaxFraction = 0.66f + (fs - 1f) * 0.22f    // 바텀시트 상한(글씨 클수록 ↑)

위계 스케일 26/24/20/18/15/13 · 디스플레이 배율 fsd = 1+(fs-1)×0.6. 상세 수치는 레포 스펙 handoff-realtime-spec.md §3.

Map · fit-bounds · 확대/축소

지도 / 마커

마커는 상수 크기(줌에 따라 커지거나 작아지지 않음). 표시할 핀들의 bounding box를 가시영역(시트 제외) 안에 맞춰 줌만 조정한다 — 우버/구글맵 스펙. 바텀시트가 내려가면 넓어진 지도영역 기준으로 재-fit.

MapFit.ktMaps SDK
// 마커는 상수 크기(줌 무관). 표시 핀 bounding box를 가시영역에 맞춰 '줌만' 조정.
val bounds = LatLngBounds.builder().apply { pins.forEach { include(it) } }.build()
val padding = with(density) { 20.dp.roundToPx() }        // 상하좌우 20dp
map.animateCamera(
  CameraUpdateFactory.newLatLngBounds(bounds, padding), // + 최대 줌 클램프
  /* durationMs = */ 440, null,
)
// 운행 시작 후 = 다음 포커스 핀 + 현재 차량 위치만.

지도 확대/축소 — 탭·버튼 토글 (드래그 아님)

기본(지도+시트) ⇄ 확대(시트 숨김·풀 지도). 프로토는 지도 탭으로 토글하지만, 실앱은 지도 탭이 네이티브 팬/핀치와 충돌 → 전용 확대/축소 버튼 권장. 지도 자체 줌은 네이티브 핀치·더블탭. (Figma ④ A 참조)

MapExpand.ktCompose
// 지도 확대/축소 = 상태 토글 (드래그 아님 — 전용 버튼 1탭).
var expanded by remember { mutableStateOf(false) }

if (!expanded) BottomSheet(state)              // 기본: 지도 + 시트
FloatingToggle(                                // 전용 확대/축소 버튼
  icon  = if (expanded) Icons.Collapse else Icons.Expand,
  label = if (expanded) "지도 축소하기" else null,
) { expanded = !expanded; performHapticFeedback() }

// 상태 바뀌면 가시영역이 달라짐 → 재-fit (마커는 상수 크기 유지)
LaunchedEffect(expanded) {
  val padding = if (expanded) fullPadding else sheetExcludedPadding
  map.animateCamera(CameraUpdateFactory.newLatLngBounds(bounds, padding), 440, null)
}
// 지도 자체 줌(핀치·더블탭)은 지도 SDK가 처리 — 위 토글과 별개.
Edge · 잊지 말 것

엣지케이스 · 안전

  • 경유 N개 — 05 drive-go ↔ 08 drive-arrived를 경유 수만큼 반복. 남은 경로 카운트는 동적.
  • 장소명 줄바꿈 = 문자 단위 (keep-all 아님). T맵 데이터엔 띄어쓰기 없는 긴 이름이 잦다 (예 “한양대학교사법대학부속고등학교”).
  • 배지 = 지도 마커 상태색 통일 — 완료=navy(물러남) · 현재 타깃=파랑(경유/출발)·빨강(도착) · 대기=navy. 리스트와 지도가 항상 같은 상태색.
  • 완료 = 물러남 — 리스트는 초록 “완료”(체크 없이), 지도핀은 opacity 50%. 현재 타깃만 시선을 독점한다.
안전 카피(필수 유지). 결제 화면의 “절대 카드/현금을 직접 받지 마세요” 문구는 어떤 경우에도 유지한다.

이 페이지의 역할

Figma는 시각·측정·토큰(Dev Mode), 프로토는 인터랙티브 동작, 이 허브는 로직·모션·코드를 담는다. 세 층이 짝을 이뤄 하나의 핸드오프를 이룬다. 코드 스니펫은 값(타이밍·이징·배율)이 프로토 확정치이며, 구조는 팀 아키텍처에 맞게 조정한다.
